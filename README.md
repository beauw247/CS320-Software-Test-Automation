Reflection
How can I ensure that my code, program, or software is functional and secure?

I ensure my code is functional by using unit testing to validate each requirement and confirm expected behavior. 
Writing tests alongside development helps catch errors early and ensures consistency. 
To support security, I enforce input validation and prevent invalid data from entering the system, which reduces 
the risk of unexpected behavior or vulnerabilities.

How do I interpret user needs and incorporate them into a program?

I interpret user needs by translating requirements directly into rules within the code. 
Constraints such as character limits, required fields, and unique identifiers are implemented as enforced conditions. 
This ensures the program behaves in a way that aligns with user expectations and business requirements.

How do I approach designing software?

I approach software design by starting with the core functionality and breaking it into smaller, manageable components. 
I build step by step, connecting classes and methods while testing along the way. This approach keeps the design organized
and makes debugging easier. Writing tests early also helps guide my design decisions and ensures the system behaves as 
intended.

Summary

These projects strengthened my understanding of how testing, automation, and design work together. 
They reinforced that writing code is only part of development, and ensuring reliability, accuracy, and maintainability is just 
as important.
